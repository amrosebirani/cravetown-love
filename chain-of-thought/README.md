# Chain of Thought Documentation

This folder contains design documents, meeting notes, analysis, and planning documents for Cravetown development.

## Folder Structure

```
chain-of-thought/
├── meetings/           # Meeting notes and discussion summaries
├── systems/            # System design documents and specifications
├── todo-lists/         # Implementation plans and progress tracking
└── analysis/           # Analysis documents and recommendations
```

## Folder Contents

### meetings/
Meeting notes capturing discussions, decisions, and action items.
- Named format: `meeting_notes_YYYY-MM-DD_participant.md` or descriptive names
- Contains discussion summaries, decisions made, and follow-up items

### systems/
Core system design documents and specifications.
- Architecture documents
- UI/UX specifications
- Feature design documents
- Implementation specifications

### todo-lists/
Planning documents and progress tracking.
- Development roadmaps
- Feature implementation checklists
- Progress tracking documents
- Task rubrics and guidelines

### analysis/
Research, analysis, and recommendations.
- Data analysis (cravings, vectors, fulfillment)
- Design recommendations
- Balancing suggestions
- Research findings

## Naming Conventions

- Use lowercase with hyphens for new files: `feature-name-type.md`
- Meeting notes: `meeting_notes_YYYY-MM-DD_topic.md`
- Design docs: `system-name-design.md` or `feature-implementation.md`
- Analysis: `topic-analysis.md` or `topic-recommendations.md`
